--------------------------------------------------------------------------------
  v1.0.0 — Initial Release
  Date: 2026-03-12
--------------------------------------------------------------------------------

  First working release of the Graph RAG knowledge base system.

  FEATURES INCLUDED:
  - Document ingestion (.txt, .md, .pdf) with chunking and embedding
  - PostgreSQL + pgvector backend via Supabase on Docker
  - Sentence-transformers (all-MiniLM-L6-v2, 384-dim) for offline embeddings
  - spaCy NER + noun phrase extraction for entity/relation graph building
  - Hybrid retrieval: vector similarity search + knowledge graph traversal
  - NetworkX in-memory DiGraph synced from PostgreSQL
  - Dual LLM provider support: LM Studio (offline) and Claude API (online)
  - MCP server (FastMCP) with 5 tools for Claude Code and LM Studio integration
    - retrieve_context
    - ingest_document
    - get_knowledge_base_stats
    - build_knowledge_graph
    - search_chunks
  - Interactive chat REPL (chat.py) with slash commands
  - CLI entry point (main.py) with 6 subcommands: ingest, build-graph,
    create-index, query, stats, models
  - Incremental graph building (tracks graph_indexed_at per chunk)
  - IVFFlat vector index support for fast similarity search
  - Context assembly with token budget management (~3000 tokens)
  - Full setup automation via install.py
  - JWT/secret key generator for Supabase auth


--------------------------------------------------------------------------------
  v1.1.0 — Performance Improvements
  Date: 2026-03-14
--------------------------------------------------------------------------------

  CHANGES:

  - Query embedding cache (embeddings/encoder.py)
    Added @lru_cache(maxsize=256) to encode_one(). Repeated queries no longer
    re-run the sentence-transformer model — result is returned instantly from
    cache on subsequent calls.

  - Graph cache with invalidation (graph/store.py)
    load_graph_from_db() now caches the NetworkX DiGraph in memory after the
    first load. Subsequent queries reuse the cached graph instead of hitting
    the database every time. Cache is automatically invalidated (reset to None)
    after any ingest or build_knowledge_graph call so the graph stays fresh.

  - Graph expansion cap (retrieval/graph_search.py)
    expand_with_graph() now stops collecting chunk IDs once 50 are reached.
    Prevents runaway expansion on dense graphs that could flood context
    assembly with hundreds of low-relevance chunks.

  - Graph-expanded chunk scoring (retrieval/hybrid.py)
    Graph-expanded results previously returned similarity=0.0, putting them
    at the bottom with no differentiation. They now receive a score derived
    from the lowest vector seed score minus a decay penalty (0.1), placing
    them just below direct vector matches. Results are sorted by similarity
    descending so context assembly receives the best chunks first.

  FILES CHANGED:
    embeddings/encoder.py
    graph/store.py
    ingestion/pipeline.py
    retrieval/graph_search.py
    retrieval/hybrid.py
    mcp_server.py


--------------------------------------------------------------------------------
  v1.2.0 — Correctness Fixes
  Date: 2026-03-14
--------------------------------------------------------------------------------

  CHANGES:

  - Claude API retry logic (llm/client.py)
    _claude_chat() now retries up to 3 times with exponential backoff (1s, 2s,
    4s), matching the existing LM Studio retry behaviour. Transient API errors
    no longer cause immediate failures. A warning is logged on each failed
    attempt and a clear error is raised if all retries are exhausted.

  - Entity deduplication by type + label (graph/extractor.py)
    Entities were previously deduplicated by lowercase label alone, causing
    unrelated entities sharing the same text (e.g. "Apple" the company and
    "apple" the concept) to merge into one node. The dedup key is now
    "{node_type}:{label.lower()}" so entities of different types are kept
    distinct. Fixed in both the spaCy path and the regex fallback.

  - Log skipped graph edges (graph/builder.py)
    When a graph edge cannot be created because a source or target node ID is
    missing, a WARNING is now logged with the edge labels and chunk ID instead
    of silently skipping. Makes graph build issues visible during debugging.

  - Configurable chars-per-token ratio (config.py, rag/context_builder.py)
    The context budget calculation previously used a hardcoded _CHARS_PER_TOKEN
    = 4, which is inaccurate for Claude (~3.5 chars/token) vs LM Studio models.
    The value is now read from CHARS_PER_TOKEN in .env (default: 4) so it can
    be tuned per provider without changing code.

  FILES CHANGED:
    llm/client.py
    graph/extractor.py
    graph/builder.py
    rag/context_builder.py
    config.py


--------------------------------------------------------------------------------
  v1.3.0 — Centralized Logging
  Date: 2026-03-14
--------------------------------------------------------------------------------

  CHANGES:

  - Centralized logging configuration (logging_config.py — new file)
    Single setup_logging(verbose, stderr_only) function replaces the duplicate
    logging.basicConfig() calls that existed in both main.py and mcp_server.py.
    All entry points import and call it once at startup. Other modules only
    call logging.getLogger(__name__) — they never configure logging themselves.

    verbose=True  → DEBUG level (enabled via -v flag in CLI)
    stderr_only=True → routes all log output to stderr (used by MCP stdio mode
                       so stdout stays clean for the MCP protocol)
    force=True is set so any accidental early basicConfig call is overridden.

  - main.py cleanup
    Removed the local _setup_logging() function. Now calls setup_logging()
    from logging_config. Removed unused `import logging` and `import json`.

  - mcp_server.py cleanup
    Replaced two separate logging.basicConfig() blocks (one for HTTP, one for
    stdio) with single setup_logging() calls. Log format is now identical to
    the CLI. stdio mode sets WARNING level after setup to suppress INFO noise
    on the protocol channel.

  - embeddings/encoder.py
    Replaced print("[encoder] ...") calls with log.info() via a proper
    logging.getLogger(__name__) logger. Messages now appear with consistent
    timestamp + level + module name format instead of a manual [encoder] prefix.

  Log format (all modules, all entry points):
    12:34:56 [INFO] embeddings.encoder: Loading embedding model: all-MiniLM-L6-v2
    12:34:56 [INFO] embeddings.encoder: Model ready (dim=384)
    12:34:56 [INFO] ingestion.pipeline: Created 42 chunks
    12:34:56 [WARNING] llm.client: Claude API attempt 1/3 failed: ... retrying in 1s

  FILES CHANGED:
    logging_config.py  (new)
    main.py
    mcp_server.py
    embeddings/encoder.py


--------------------------------------------------------------------------------
  v1.4.0 — Typed Retrieval Results
  Date: 2026-03-14
--------------------------------------------------------------------------------

  CHANGES:

  - RetrievalResult dataclass (db/models.py)
    Added RetrievalResult(chunk_id, document_id, content, similarity, source)
    to the existing models module. Replaces the untyped plain dicts that were
    previously passed through the entire retrieval pipeline.

  - match_chunks returns RetrievalResult (db/queries.py)
    The DB query function now constructs RetrievalResult objects directly from
    row data instead of building raw dicts.

  - vector_search returns list[RetrievalResult] (retrieval/vector_search.py)
    Return type updated; no logic change.

  - hybrid retrieve returns list[RetrievalResult] (retrieval/hybrid.py)
    Graph-expanded chunks are now wrapped in RetrievalResult with source="graph"
    instead of being appended as dicts. Attribute access replaces all dict key
    lookups (r.chunk_id, r.similarity, r.source).

  - context_builder accepts list[RetrievalResult] (rag/context_builder.py)
    All r.get() dict calls replaced with direct attribute access.

  - pipeline sources use r.chunk_id (rag/pipeline.py)
    r.get("chunk_id", "") replaced with r.chunk_id.

  - mcp_server attribute access (mcp_server.py)
    r.get("source") and r.get("similarity") replaced with r.source and
    r.similarity.

  FILES CHANGED:
    db/models.py
    db/queries.py
    retrieval/vector_search.py
    retrieval/hybrid.py
    rag/context_builder.py
    rag/pipeline.py
    mcp_server.py
